The Adventures of Mr. Turtle

Roles: Coder, Developer

Summary: I created a video game remake of Frogger.

Tools Used: Processing 3.2

_____

The Challenge

This was an individual project that I worked on during my free time and outside of school. The reason for creating a video game was through the inspiration of a recent course that I had taken at school. In my previous university term, I had taken a computer science course that taught me how to code in Java and I wanted to use this knowledge that Iâ€™d just learned to create a game because of my passion for gaming. I created this game in reference to my favourite childhood game Frogger.

The Process

To build this game, I used the program called Processing and designed all my game elements throughout this program. For my design process, I followed a similar layout to Frogger, however there are some different features such as instead of a frog the main character is a turtle, and instead of jumping on the logs to pass the river, you must avoid them. When creating my game pieces, I went the through the process of designing my game characters first and then coding them into the game afterwards. I would research and brainstorm how my characters would look and then design them on paper to visually display my ideas. I would then take these designs into my program and code them, and this allowed me to see how the full character would interact in my game rather than just outlines of my designs. This also allowed me to focus on one segment of my program at a time to make sure it was running properly before moving onto the next element.

Upon completion of my game, I went to a couple of my peers at school and had them test out my game to obtain their input. They informed me that my game was really challenging, so I went back into my code and reduced the speed of the enemies to make it more playable. Through this process, I was able to acquire feedback that allowed me to make my game more user-friendly. Overall, I believe I was able to successfully create an interactive video game for my personal usage as well as for my friends and family to use.

Skills Learned

A skill that I was able to demonstrate in the production of this game was my problem-solving skills. Throughout the creation of this game, I would constantly run into problems specifically with my code not working. Some of these problems included; game characters not appearing on the screen, hit detection not registering, or characters moving into off limit areas. Whenever a problem occurred with my game, the first thing that I would do would be to define and identify the root cause. I would do things such as restarting the game and playing it until the problem occurred again or freezing sections of my code and running diagnostic tests to determine if they were working properly. Once I had identified the problem, I would then try to come up with various solutions, such as using a different formula, restructuring my code, or changing values. After I had solved the problem, I would test run the game a couple of times to ensure it was running properly before moving on with the coding process.
